Js¶

Desde el Comienzo, el Soporte de CakePHP párr Javascript ha Sido estafa Prototyp / Scriptaculous. MIENTRAS Seguimos pensando Que ESTAS hijo Unas Excelentes Bibliotecas de Javascript, la comunidad ha Estado Pidiendo Soporte HACIA Otras Bibliotecas. En Lugar De tirar Prototype es a favor de Otra biblioteca de Javascript. HEMOS Puntos Creado ONU Adaptador basado en el los ayudantes, e incluimos 3 De Las bilbiotecas mas solicitadas. Prototype / Scriptaculous, Mootools / Mootools-más, y jQuery / jQuery UI. Y mientras que la API no es tan extensa como el AjaxHelper anterior consideramos que la solución basada adaptador permite una solución más extensible dando a los desarrolladores el poder y la flexibilidad que necesitan para hacer frente a sus necesidades de aplicación específicas.

Motores de JavaScript son la columna vertebral de la nueva JsHelper. Un motor de Javascript traduce un elemento Javascript abstracto en concreto de código Javascript específico a la biblioteca Javascript está utilizando. Además crean un sistema extensible para que otros la utilicen.

El uso de un motor específico Javascript¶

En primer lugar descargar su biblioteca de Javascript preferido y colocarla en app / webroot / js

A continuación, debe incluir la biblioteca en su página. Para incluirlo en todas las páginas, añadir la siguiente línea a la sección <head> de app / views / layouts / default.ctp (copiar este archivo desde cake / libs / view / layouts / default.ctp si no ha creado el suyo propio) .

echo  $ this -> Html -> guión ( 'jquery' );  / / Incluir la biblioteca jQuery

Reemplace jQuery con el nombre de su archivo de biblioteca (. js se agregarán a su nombre).

Por defecto está en caché, y usted debe imprimir explícitamente la caché. Para ello, al final de cada página, incluya esta línea justo antes de que el final </ body> etiqueta

echo  $ this -> Js -> writeBuffer ();  / / escribir archivos de comandos en caché

Usted debe incluir a la biblioteca en su página e imprimir la memoria caché para el ayudante de funcionar.

Javascript selección del motor se declara cuando se incluye la ayuda en su controlador.

var  $ helpers  =  array ( 'Js'  =>  array ( 'Jquery' ));

Lo anterior sería utilizar el motor de Jquery en los casos de JsHelper en sus puntos de vista. Si no se declara un motor específico, el motor de jQuery se utilizará como predeterminado. Como se ha mencionado antes, hay tres motores implementados en el núcleo, pero alentamos a la comunidad para ampliar la compatibilidad de la biblioteca.

Usando jQuery con otras bibliotecas¶

La biblioteca jQuery, y prácticamente todos sus plugins se ven limitados en el espacio de nombres de jQuery. Como regla general, los objetos "globales" se almacenan en el interior del espacio de nombres de jQuery, así, por lo que no deben recibir un choque entre jQuery y cualquier otra biblioteca (como Prototype, Mootools, YUI o).

Dicho esto, hay una advertencia: De forma predeterminada, jQuery utiliza "$" como una abreviatura de "jQuery"

Para anular el acceso directo "$", utilice la variable jQueryObject.

$ This -> Js -> JqueryEngine -> jQueryObject  =  '$ j' ; 
print  $ this -> Html -> ScriptBlock ( 'var $ j = jQuery.noConflict ();' , 
    array ( 'inline'  =>  true )) ;  / / Dile a jQuery para pasar al modo noconflict

Usando el JsHelper dentro customHelpers¶

Declarar el JsHelper en ayudantes $ array en su customHelper.

var  $ helpers  =  array ( 'Js' );

No es posible declarar un motor de javascript dentro de un helper personalizado. Hacer eso no tendrá ningún efecto.

Si usted está dispuesto a usar otro motor de javascript de la predeterminada, realizar la configuración auxiliar en el controlador de la siguiente manera:

var  $ helpers  =  array ( 
    'Js'  =>  array ( 'Prototype' ), 
    'CustomHelper' 
);

Asegúrese de declarar el JsHelper y su motor en la parte superior del ayudantes $ matriz en el controlador.

El javascript motor seleccionado puede desaparecer (reemplazado por el valor predeterminado) desde el objeto jsHelper en su ayudante, si se olvida de hacerlo y obtendrá un código que no se ajuste a tu biblioteca de Javascript.

Creación de un motor de Javascript¶

Ayudantes motor de JavaScript siguen las convenciones normales de ayuda, con algunas restricciones adicionales. Ellos deben tener el motor sufijo. DojoHelper no es bueno, DojoEngineHelper es correcta. Además, deben extender JsBaseEngineHelper con el fin de aprovechar al máximo la nueva API.

Javascript uso del motor¶

El JsHelper proporciona algunos métodos, y actúa como una fachada para el ayudante del motor. Usted no debe acceder directamente al ayudante de motor, excepto en raras ocasiones. Uso de las características de la fachada de la JsHelper le permite aprovechar el almacenamiento en búfer y las características de encadenamiento de métodos incorporados; (Método de encadenamiento sólo funciona en PHP5).

El JsHelper por buffers por defecto casi todo el código de secuencia de comandos generada, lo que le permite recoger los scripts en todo el juicio, los elementos y el diseño, y la muestran en un solo lugar. Salida de secuencias de comandos almacenados en el búfer se hace con $ this-> Js-> writeBuffer (); esto devolverá el contenido del búfer en una etiqueta de script. Puede deshabilitar el almacenamiento en búfer por mayor con el $ bufferScripts propiedad o el establecimiento de búfer => false en los métodos que toman $ options .

Como la mayoría de los métodos en Javascript comienzan con una selección de elementos en el DOM, $ this-> Js-> get () devuelve un $ this, lo que le permite a la cadena de los métodos que utilizan la selección. Encadenamiento Método permite escribir más corto, el código más expresiva. Cabe señalar que el método de encadenamiento no va a trabajar en PHP4.

$ This -> Js -> obtener ( '# foo' ) -> evento ( 'click' ,  $ eventCode );

Es un ejemplo de método de encadenamiento. Método de encadenamiento no es posible en PHP4 y la muestra anterior se escribiría así:

$ This -> Js -> get ( '# foo' ); 
$ this -> Js -> evento ( 'click' ,  $ eventCode );

Las opciones comunes¶

En un intento por simplificar el desarrollo donde las bibliotecas Js pueden cambiar, un conjunto común de opciones es apoyado por JsHelper , estas opciones comunes se asignarán a las opciones específicas de la biblioteca interna. Si usted no planea cambiarse librerías Javascript, cada biblioteca también es compatible con la totalidad de sus devoluciones de llamada y las opciones nativas.

Embalaje de devolución de llamada¶

Por defecto todas las opciones de devolución de llamada se envuelven con la una función anónima con los argumentos correctos. Puede desactivar este comportamiento mediante el suministro de la wrapCallbacks = falsa en su gama de opciones.

Trabajar con scripts de búfer¶

Un inconveniente de la aplicación anterior de características de tipo 'Ajax' fue la dispersión de etiquetas script en todo el documento, y la incapacidad para amortiguar los scripts agregados por los elementos en el diseño. El nuevo JsHelper si se utiliza correctamente evita esas dos cuestiones. Se recomienda que usted coloca $ this-> Js-> writeBuffer () en la parte inferior de su archivo de diseño por encima de la </ body> etiqueta. Esto permitirá que todos los scripts generados en los elementos de diseño se impriman en un solo lugar. Cabe señalar que los guiones tamponadas son manejados por separado de los archivos de script incluidas.

writeBuffer ($ opciones = array ())

Escribe todas Javascript generado hasta ahora un bloque de código o los almacena en caché en un archivo y devuelve un script vinculado.

Opciones

  • en línea - Se establece en true para que la salida guiones como un bloque en línea script si caché también es cierto, se generará una etiqueta de vínculo de script. (Por defecto true)
  • cache - Se establece en true para que los scripts en caché en un archivo y vinculados en (false por defecto)
  • claro - Se establece en false para evitar que la memoria caché de la escritura se borre (por defecto true)
  • onDomReady - guiones wrap en caché en caso domready (por defecto true)
  • seguro - si se genera un bloque de línea debería ser envuelto en <! [CDATA [...]]> (predeterminado true)

Creación de un archivo de caché con writeBuffer () requiere que webroot / js ser escribible mundo y permite a un navegador para almacenar en caché los recursos de script generado para cualquier página.

tampón ($ contenido)

Añadir $ contenido a la memoria intermedia de la escritura interna.

GetBuffer ($ clear = true)

Consiga el contenido del búfer en uso. Pase en false para no borrar el búfer al mismo tiempo.

Métodos de tamponamiento que normalmente no están amortiguadas

Algunos métodos de los ayudantes se almacenan de forma predeterminada. Los motores de amortiguar los métodos siguientes de manera predeterminada:

  • evento
  • sortable
  • arrastrar
  • soltar
  • deslizador

Además, puede obligar a ningún otro método en JsHelper utilizar el almacenamiento en búfer. Añadiendo un boolean hasta el final de los argumentos que puede obligar a otros métodos para entrar en el búfer. Por ejemplo, la each () método normalmente no búfer.

$ This -> Js -> cada uno ( 'alert ("! whoa");' ,  verdadero );

Lo anterior obligaría al each () método para utilizar la memoria intermedia. Por el contrario, si usted quiere un método que el tampón no búfer, se puede pasar un falso como el último argumento.

$ This -> Js -> evento ( 'click' ,  'alert ("! whoa");' ,  false );

Esto obligaría a la función de eventos que normalmente amortigua para volver su resultado.

Métodos¶

Los núcleos motores de JavaScript proporcionan las mismas características en todas las bibliotecas, también hay un subconjunto de las opciones comunes que se traducen en opciones específicas de la biblioteca. Esto se hace para proporcionar a los desarrolladores de extremo con unificada como una API como sea posible. La siguiente lista de métodos son compatibles con todos los motores incluidos en el núcleo de CakePHP. Cada vez que vea listas separadas para Opciones y eventos Opciones ambos conjuntos de parámetros se suministran en la $ options array para el método.

objeto ($ data, $ options = array ())

Convierte valores en JSON. Hay algunas diferencias entre este método y JavascriptHelper :: () del objeto. Lo más notable es que no hay potencialidad para stringKeys o q opciones que se encuentran en el JavascriptHelper. Además $ this-> Js-> () del objeto; no pueden hacer etiquetas de script.

Opciones:

  • prefijo - String antepuesto a los datos devueltos.
  • Postfix - String anexa a los datos devueltos.

Ejemplo Uso:

$ Json  =  $ this -> Js -> objeto ( $ data );

se pueden ordenar ($ options = array ())

Ordenable genera un fragmento de código JavaScript para realizar un conjunto de elementos (por lo general una lista) de arrastrar y soltar se puede ordenar.

Las opciones normalizadas son:

Opciones

  • contención - Contenedor de acción de movimiento
  • manejar - Selector de manejar elemento. Sólo este elemento se iniciará acción de ordenación.
  • volver - Si desea utilizar o no un efecto a mover clasificable en su posición final.
  • opacidad - Opacidad del marcador de posición
  • distancia - Distancia a sortable debe ser arrastrado antes de clasificarlas como titular.

Opciones de eventos

  • inicio - Evento disparó al ordenar aperturas
  • tipo - Evento disparó durante la clasificación
  • complete - Evento disparó al ordenar la ultima.

Otras opciones son compatibles con cada biblioteca de Javascript, y usted debe comprobar la documentación de la biblioteca javascript para obtener información más detallada acerca de sus opciones y parámetros.

Ejemplo del uso:

$this -> Js -> get ( '#my-list' );

        'distance'  =>  5 ,

solicitud ($ url, $ options = array ())

Generar un fragmento de código javascript para crear una XmlHttpRequest o solicitud 'AJAX'.

Opciones de eventos

  • completar - Devolución de llamada a disparar contra completa.
  • éxito - Devolución de llamada al fuego en caso de éxito.
  • antes - Devolución de llamada al fuego a petición de inicialización.
  • error - Devolución de llamada al fuego en caso de fallo petición.

Opciones

  • Tipo - Tipo de datos para la respuesta. 'Json' y 'html' son compatibles. El valor predeterminado es HTML para la mayoría de las bibliotecas.
  • evalScripts - Sea o no <script> deben eval'ed.
  • dataExpression - Si la información se tratará clave como llamada de retorno. Útil para el suministro de $ options ['datos'] como otra expresión Javascript.

Ejemplo de uso

get ($ selector)

Ajuste la "selección" interna a un selector CSS. La selección activa se utiliza en las operaciones posteriores hasta que se realice una nueva selección.

$ This -> Js -> get ( '# element' );

El JsHelper ahora consultará todos los demás métodos de elementos basados ​​en la selección de # element . Para cambiar la selección activa, llame a get () de nuevo con un nuevo elemento.

arrastre ($ opciones = array ())

Hacer un elemento arrastrable.

Opciones

  • manejar - selector en la empuñadura.
  • snapGrid - La cuadrícula de píxeles que el movimiento se ajusta a, una matriz (x, y)
  • contenedor - El elemento que actúa como un cuadro de delimitación para el elemento arrastrable.

Opciones de eventos

  • inicio - Evento que se activa cuando se inicia el arrastre
  • drag - Evento disparó en cada paso del arrastre
  • stop - Evento disparó al arrastrar paradas (liberación del ratón)

Ejemplo de uso

$this -> Js -> get ( '#element' ); 
$this -> Js -> drag ( array ( 
    'container'  =>  '#content' , 
    'start'  =>  'onStart' , 
    'drag'  =>  'onDrag' ,

));

Si se utiliza el motor de jQuery se añadiría el siguiente código en el búfer.

$ ( "#element" ) . draggable ({ containment : "#content" ,  drag : onDrag ,  grid : [ 10 , 10 ],  start : onStart ,  stop : onStop });

gota ($ options = array ())

Haga un elemento aceptar elementos arrastrables y actuar como zona de saltos para los elementos arrastrados.

Opciones

  • aceptar - Selector de elementos de esta lanzables aceptará.
  • hoverclass - Class para agregar a Soltable cuando un arrastrable ha terminado.

Opciones de eventos

  • drop - Evento desencadena cuando un elemento se deja caer en la zona de descenso.
  • libración - Evento disparó cuando un arrastre entra en una zona de arrastre.
  • dejar - Evento que se activa cuando una carga se retira de una zona de colocación sin que se cayó.

Ejemplo de uso

$this -> Js -> get ( '#element' ); 
$this -> Js -> drop ( array (

    'leave'  =>  'onExit' ,

enlace ($ title, $ url = null, $ options = array ())

Crear un elemento de anclaje HTML que tiene un evento de clic con destino a la misma. Las opciones pueden incluir tanto los de HtmlHelper :: link () y JsBaseEngine :: request (), JsBaseEngine :: caso (); $ htmlAttributes se utiliza para especificar opciones adicionales que se supone que se adjunta al elemento de anclaje generado. Si una opción no es parte de los atributos estándar o $ htmlAttributes será pasado a petición () como una opción. Si no se suministra un id, un generado de forma aleatoria se creará uno para cada enlace generado.

Opciones

  • Identificación - utilizar un identificador personalizado.
  • htmlAttributes - htmlAttributes no estándar. Atributos estándar son de clase, id, rel, título, escape, onblur y onfocus.
  • búfer - Desactivar el buffering y devolver una etiqueta script además del enlace.

Ejemplo de uso

Serializar el formulario adjunto a $ selector. Pase cierto por $ ISForm si la selección actual es un elemento de formulario. Convierte la forma o el elemento de formulario que se adjunta a la selección actual en un objeto de cadena / json (dependiendo de la implementación de bibliotecas) para su uso con las operaciones XHR.

Opciones

Paginación Ajax¶

Al igual que el Ajax de la paginación en 1.2, puede utilizar el JsHelper para manejar la creación de enlaces de paginación Ajax en lugar de enlaces HTML plano.

Haciendo Links Ajax¶

Para poder crear enlaces ajax debe incluir la librería Javascript que coincide con el adaptador que está utilizando con JsHelper . Por defecto, el JsHelper utiliza jQuery. Así que en su diseño incluir jQuery (o cualquier biblioteca que está utilizando). Asimismo, asegúrese de incluir RequestHandlerComponent en sus componentes. Agregue lo siguiente a su controlador:

var  $ components  =  array ( 'RequestHandler' ); 
var  $ helpers  =  array ( 'Js' );

Siguiente enlace en la biblioteca javascript que desea utilizar. Para este ejemplo vamos a usar jQuery.

echo  $ this -> Html -> guión ( 'jquery' );

Similar a 1.2 Usted necesita decir el PaginatorHelper que desea hacer enlaces Javascript mejoradas en lugar de los llanos del HTML. Para ello utiliza las opciones ()

$ This -> Paginator -> Opciones ( array ( 
    'update'  =>  '# contenido' , 
    'evalScripts'  =>  cierto 
));

El PaginatorHelper ahora sabe que hacer con javascript mejorado los vínculos, y que esos vínculos debe actualizar la # content elemento. Por supuesto, este elemento debe existir, y muchas veces se desee para envolver $ content_for_layout con un div que coincida con el ID utilizado para la actualización de la opción. También debe establecer evalScripts en true si está utilizando los adaptadores Mootools o Prototype, sin evalScripts estas bibliotecas no podrán peticiones cadena juntos. El indicador de la opción no es apoyada por JsHelper y será ignorado.

A continuación, crea todos los enlaces que sean necesarios para sus funciones de paginación. Desde el JsHelper amortigua automáticamente todo el contenido script generado para reducir el número de <script> etiquetas en el código fuente que debe llamar a escribir el buffer de salida. En la parte inferior de tu archivo de vista. Asegúrese de incluir:

echo  $ this -> Js -> writeBuffer ();

Si omite esta usted no será capaz de ajax cadena de enlaces de paginación. Cuando se escribe el buffer, sino que también está despejado, por lo que no tiene que preocuparse acerca de la misma se emita Javascript dos veces.

Adición de efectos y transiciones¶

Dado que el indicador ya no es compatible, debe agregar cualquier efecto indicador por sí mismo.

< HTML > 
    < cabeza > 
        <? php  echo  $ this -> Html -> guión ( 'jquery' );  ?>
        . / / Más cosas aquí 
    </ head> 
    <body> 
    <div id="content"> 
        <php?  echo  $ content_for_layout ;  ?>
    </ Div> 
    <? php  echo  $ this -> Html -> imagen ( 'indicator.gif' ,  array ( 'id'  =>  'ocupado en indicadores' ));  ?>
    </ Body> 
</ html>

Recuerde que debe colocar el archivo dentro indicator.gif aplicación Webroot carpeta / / img. Usted puede ver una situación donde el indicator.gif muestra inmediatamente después de la carga de la página. Es necesario poner en este css # ocupada indicador { display: none; } en el archivo principal css.

Con la disposición anterior, hemos incluido un archivo de imagen indicador, que mostrará una animación indicador de actividad que vamos a mostrar y ocultar con el JsHelper . Para hacer eso necesitamos actualizar nuestras opciones () función.

$this -> Paginator -> options ( array ( 
    'update'  =>  '#content' , 
    'evalScripts'  =>  true , 
    'before'  =>  $this -> Js -> get ( '#busy-indicator' ) -> effect ( 'fadeIn' ,  array ( 'buffer'  =>  false )), 
    'complete'  =>  $this -> Js -> get ( '#busy-indicator' ) -> effect ( 'fadeOut' ,  array ( 'buffer'  =>  false )), 
));

Esto mostrar / ocultar el elemento ocupado en indicadores antes y después de la # contenido se actualiza div. Aunque el indicador se ha eliminado, las nuevas características ofrecidas por JsHelper permiten más control y efectos más complejos que se creen.
