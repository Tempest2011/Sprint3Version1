Otra cosa duda recomiendo es comprobar que la carga de sus JS en una vista es 100% OK con todo.

NO use $ this-> Html-> secuencia de comandos si no está seguro acerca de:

--- La dependencia de secuencias de comandos en sí

--- La dependencia de algunos plugins en otras escrituras

--- La cantidad de diferentes diseños y / o puntos de vista que desee utilizar en el futuro.

Por ejemplo, algunos plugins utilizan jQuery jQuery pero requieren que se carga antes que el propio plugin.

Así que si usted pone jQuery en una vista o el diseño, JS cargas del plugin antes de jQuery y el plugin no funcionen. Si desea asegurarse de que algunas cargas de guión primero, puede intentar lo siguiente:

en su aplicación / View / Helper / AppHelper.php:

clase  AppHelper  extiende  Helper  { 
    público $ helpers = array ( 'html' ,  'Sesión' );

    público  la función beforeRender ( $ viewFile ) { 
            echo $ this -> Html -> guión ( 'jquery' , array ( 'inline' => true )); 
        } 
}

Por lo tanto usted no tiene que incluir jQuery en múltiples puntos de vista y se carga por primera vez.

$ Scripts_for_layout en un diseño es un marcador de posición para todas las secuencias de comandos incluidas en las vistas, así que si incluirlo o no, no tendrá ningún efecto si carga los scripts directamente en el default.ctp .

Dicho esto, añadiendo la extensión o dejándolo fuera del $ this-> Html-> guión () llamada no debe hacer una diferencia. ¿Cuál podría ser el caso es que el HtmlHelper no se llama correctamente (se llama por defecto en Cake 2.0), pero usted debe estar recibiendo mensajes de error. ¿Está de depuración establece en 2 en / app / config / core.php ?
